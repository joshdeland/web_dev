Component Explanation

My component is a combination of a filter menu and a product card generator. The function takes two input parameters: a list of products and a list of filters. For each product in the product list, a product card is generated. All product in the product list have three required fields: a thumbnail image, a product name, and a price. As long as those requirements are met, any other attributes are optional and the product card can be displayed. The filters within the filter menu are generated from a filter list. For all filters, the user is required to have these fields: type, title, and attribute. There are two types of filters: button filters and slider filters. Button filters also require a 'values' field, which is a list of all values pertaining to the attribute that the user wants to be filtered. For slider filters, they require a min, max, value, and direction field. Min and max define the range of values that can be slid between, while value is default value when the page is loaded, and changes when the user interacts with the slider. The direction denotes if products being displayed will have an attribute value greater than or less than the current slider value.
